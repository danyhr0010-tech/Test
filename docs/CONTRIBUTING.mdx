# Guía de Contribución

Gracias por tu interés en contribuir a este proyecto. Esta guía te ayudará a hacer contribuciones de manera efectiva.

## Cómo Contribuir

### Reportar Bugs

Si encuentras un bug, por favor abre un issue incluyendo:

- Descripción clara del problema
- Pasos para reproducirlo
- Comportamiento esperado vs. comportamiento actual
- Versión de Python que estás usando
- Cualquier mensaje de error relevante

### Sugerir Mejoras

Para sugerir nuevas características:

1. Verifica que no exista un issue similar
2. Abre un nuevo issue describiendo la característica
3. Explica por qué sería útil
4. Proporciona ejemplos de uso si es posible

### Pull Requests

1. Haz fork del repositorio
2. Crea una rama desde `main`:
   ```bash
   git checkout -b feature/nombre-descriptivo
   ```
3. Realiza tus cambios siguiendo los estándares de código
4. Asegúrate de que todos los tests pasen:
   ```bash
   pytest
   ```
5. Agrega tests para tu nueva funcionalidad
6. Actualiza la documentación si es necesario
7. Commit con mensajes descriptivos
8. Push a tu fork
9. Abre un Pull Request

## Estándares de Código

### Estilo

- Sigue PEP 8 para estilo de código Python
- Usa type hints en todas las funciones
- Longitud máxima de línea: 88 caracteres
- Usa comillas dobles para strings

### Documentación

- Todas las funciones públicas deben tener docstrings
- Formato de docstrings: Google Style
- Incluye ejemplos de uso cuando sea apropiado

### Tests

- Escribe tests para toda nueva funcionalidad
- Mantén la cobertura de código por encima del 80%
- Usa nombres descriptivos para los tests
- Organiza tests en clases cuando sea apropiado

## Proceso de Revisión

1. Un mantenedor revisará tu PR
2. Puede solicitar cambios o mejoras
3. Una vez aprobado, se hará merge a main
4. Tu contribución será reconocida en los release notes

## Código de Conducta

- Sé respetuoso con otros contribuidores
- Acepta críticas constructivas
- Enfócate en lo que es mejor para el proyecto
- Muestra empatía hacia otros miembros de la comunidad

## Preguntas

Si tienes dudas, no dudes en:
- Abrir un issue con la etiqueta "question"
- Contactar a los mantenedores

Gracias por contribuir!
