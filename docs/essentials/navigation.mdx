## Overview

Proper navigation structure helps users find what they need quickly and succeed with your product.

## Navigation Structure

Organize your documentation pages in a logical hierarchy.

### Main Sections

Common navigation patterns:

- **Getting Started**: Quickstart guides and setup instructions
- **Essentials**: Core concepts and fundamental features
- **API Reference**: Technical API documentation
- **Guides**: Step-by-step tutorials

## Configuration

Define your navigation structure in your configuration file:

```json
{
  "navigation": [
    {
      "group": "Getting Started",
      "pages": ["quickstart", "development"]
    },
    {
      "group": "API Documentation",
      "pages": ["api-reference/introduction"]
    }
  ]
}
```

## Best Practices

- Keep navigation depth to 2-3 levels maximum
- Use clear, descriptive labels
- Group related content together
- Put most important content first

<Note>
The navigation order determines how users discover your content.
</Note>
